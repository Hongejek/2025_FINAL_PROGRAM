# 是不是20%人发出了80%的声音？——对与一些热点事件相关的视频、文章评论区的分析

**本项目以部分热点事件相关的视频与文章为研究对象，通过对其评论区数据进行收集、整理与统计分析，尝试回答：是否存在少数用户贡献了大部分讨论内容的现象，并刻画评论区参与用户的整体特征。**

整体研究遵循“**数据获取 → 数据清洗与结构化 → 描述性统计与可视化分析**”的基本方法论路径，侧重于评论行为的总体分布特征，而非对个体用户进行追踪或画像。

**本项目仅用于学习与研究目的，禁止用于任何商业或违法用途。**

* **1.爬取视频基本信息**
  相关文件：[video_crawler.py](video_crawler.py)与[bv_list.txt](bv_list.txt)
  将想要爬取的视频BV号一行一个，放在*bv_list.txt*文件中，运行*video_crawler.py*文件,它会为每个视频生成一个包含播放量，弹幕数，投币数，点赞数，收藏量，转发数等信息的csv格式文件，放在[data\video*](data\video)目录下。
* **2.爬取评论区信息**
  相关文件：[comment_crawler.py](comment_crawler.py), [bv_cookie.txt](bv_cookie.txt)与[bv_list.txt](bv_list.txt)

#### 绘制图表

相关文件：[diagram_drawing.py](diagram_drawing.py),[data](data),[chart\image](chart\image)

***diagram_drawing.py***中共有9个函数，对应9种不同的图。可在main函数中选择运行的函数，绘制需要的图。

* ***analyze_video_engagement*---*analyze_video_average_engagement***
  分别为*data\video*中的所有以**_视频.csv**结尾的视频数据都单独绘制一张用户参与度的表---为*data\video*中的所有以**_视频.csv**结尾的视频数据的用户参与度取平均值绘制一张表。
* **函数2-7**
  为*data\comment*中的 _***一个***_ 以 **_评论.csv** 结尾的评论数据绘制相应的图片（具体请看*diagram_drawing.py*中的注释和下面"**分析发表评论的用户总体情况**"部分）。
* ***analyze_keywords_and_sentiment***
  为*data\comment*中的 _***一个***_ 以 **_评论.csv** 结尾的评论数据绘制评论关键词及情感趋势分析图表。会为评论区出现次数前10的词绘制一个饼状图，为各个时间段出现次数前5的词绘制一个折线图。

## 1. 统计项目

围绕评论区讨论结构与参与情况，主要统计以下几类指标：

* **文章、视频基本信息**：包括阅读量、播放量、点赞数、评论总数、转发数等，用于衡量内容的整体传播与互动规模。
* **用户的基本信息**：包括用户昵称（ID）、性别、注册时间等公开字段；在后续分析中仅用于统计层面，对可能涉及隐私的信息进行脱敏或不予使用。
* **评论的大致内容**：提取评论中与文章或视频主题相关的关键词，并结合情感词典，对评论的整体情感倾向进行概括性分析。
* **评论发出的时间**：结合内容发布时间，分析评论数量在不同时间段内的分布情况，用以刻画讨论热度的变化过程。

## 2. 数据获取与整理

***对于B站参与特殊活动（如拜年纪），动漫、电影、电视剧等这一类特殊的视频，它们的HTML界面完全不同，管控也更加严格，甚至其中还有一些付费内容。考虑伦理，法律法规，B站规定、底线以及我自身的时间与能力限制，故不会考虑获取这些数据。***

**数据获取时间（由于不同视频爬取时间不同而有短暂差异）：*2025年12月23日至2025年12月28日*。在此之后的数据没有记录。**

### 文章、视频基本信息获取

**B 站视频基本信息获取**：对应脚本为：[video_crawler.py](video_crawler.py)。

在该部分的数据采集中，主要采取以下方法以保证稳定性与数据规范性：

* **模拟正常访问行为**：设置合理的请求间隔（如每次请求后休眠 1–3 秒），并使用常见浏览器的 User-Agent 以减少异常访问风险。
* **准备有效的 Cookies（如有需要）**：在接口需要登录态时使用有效 Cookie，但避免不必要的频繁请求。
* **规范数据格式**：将获取到的视频信息统一保存为 CSV 文件，便于后续使用 pandas 等工具进行处理与分析。
* **添加异常情况处理**：对网络请求失败、JSON 解析错误及接口异常返回进行捕获，避免程序中断或生成不完整数据。

对于需要进行分析的的 CSV 文件统一存放于 [data\video](data\video) 目录下，否则放请存放在[数据样本\视频基本信息](数据样本\视频基本信息)目录下。

### 评论区信息获取

**B 站视频评论获取**：对应脚本为：[comment_crawler.py](comment_crawler.py)。

**获取的评论条数与视频总评论有关：如视频总评论不超过17万条，则尽量爬取全部评论，如果超过17万，则只爬取前17万条评论。**

该部分是本项目的核心数据来源，同时也是平台风控较为严格的环节，因此在设计上更侧重稳定性与可控性。

在评论数据获取过程中，主要进行了以下处理：

* **去除不必要或敏感的信息**：在保留评论内容、发布时间、IP 属地等分析所需字段的前提下，去除与研究目的关联度较低或可能涉及隐私的字段，如头像链接、个性签名等。
* **规范数据格式**：将评论内容、时间等字段统一整理并导出为 CSV 文件，方便后续统计与文本分析。
* **添加异常情况处理**：对网络异常、JSON 解析失败以及接口返回错误进行检测，确保数据获取过程可中断、可恢复。
* **避免风控**：

  * **风控产生的原因说明**：B 站对不同类型的数据接口采用了不同等级的风控策略。视频的播放量、点赞数、评论总数等视频基本信息，主要用于页面展示和推荐计算，属于高频、低敏感度的公开数据，其接口通常允许在较宽松的访问条件下被获取，因此在合理控制请求频率的情况下不易触发风控。相比之下，评论区数据涉及用户表达内容、互动关系及时间序列信息，属于平台重点保护的数据类型，对应接口的风控等级明显更高。在短时间内、以非浏览器环境或异常网络环境对评论接口进行批量访问时，容易被判定为异常行为，从而触发访问限制。**B站现在已经对弹幕的请求接口强制要求WBI签名，对于评论请求接口，也已经在进行更新并处于过渡期：兼容旧接口的同时使用WBI签名。在(2026年起)以后如果B站的评论请求接口强制要求WBI签名，本项目*comment_crawler.py*代码将不再适用，请注意更新。**
  * **遵循平台访问节奏**：控制评论接口的请求频率，避免高并发或短时间内大量翻页请求，主评论与分页请求之间设置合理延时。
  * **模拟真实浏览器环境**：在请求层面尽量保持与浏览器一致的行为特征，如使用常见 User-Agent、完整请求头，并通过模拟浏览器 TLS 指纹的方式减少异常客户端特征。
  * **合理使用登录态 Cookie**：仅在必要时使用 Cookie 访问评论接口，避免频繁更换 Cookie 或在异常环境下反复请求高风控等级接口。
  * **避免异常访问环境**：评论接口对访问环境较为敏感，尽量在账号常用地区的网络环境下运行脚本，避免在海外 IP 或频繁切换网络环境时进行大规模数据抓取。
  * **设置异常与冷却处理机制**：当接口返回访问受限（如 `-403`）时，停止脚本请求，进入冷却期，仅使用浏览器进行正常访问，避免持续请求导致风控等级进一步提升。

**替代方案**：在 API 接口受限时，可以使用[Edge扩展：获取B站评论](https://github.com/1dyer/bilibili-comments-extension.git)或[已有的B站评论分析工具](https://github.com/sansan0/bilibili-comment-analyzer)作为评论数据采集的补充方案(*如过觉得调试comment_crawler.py太麻烦强烈推荐直接选择这两个方法，但得到的csv文件要保证命名符合本项目规范，表头等内容完整，能够支持绘图，放在下面的目录中，diagram_drawing.py会从下面的目录读取csv文件*)。

对于需要进行分析的的 CSV 文件统一存放于 [data\comment](data\comment) 目录下，否则放请存放在[数据样本\评论区信息](数据样本\评论区信息)目录下。

## 4. 结果分析

依据得到的数据以及绘制的图片，我们可以对一个视频的评论区的情况进行大致的分析

### 视频基本信息与用户参与度

**对新闻，日常生活，文娱，游戏类型共40余个评论数较多，播放量较高的典型视频进行分析，得到的结果如下**：

- 1.新闻类视频用户评论数普遍较低，参与热情偏低，对于评论较多的视频，也是由于播放量较高而被拉高；但平均点赞数相对其他类型视频非常高，可认为是原意了解的很多，即使是路人也大都原意留意。
  ![alx text](chart\结果分析部分\视频基础信息\新闻_视频基础信息.png)
- 2.游戏，直播类视频点赞数较少，评论数较高，可以认为是路人较少而核心关注用户较多。
  ![alx text](chart\结果分析部分\视频基础信息\游戏_视频基础信息.png)
- 3.日常生活类视频（展示做饭，旅游，工作，学生生活，电脑等工具质量，宠物，生活妙招这类分享自己感受的视频）参与度是所有类视频最低的，评论更是低至只有播放量的0.1%，反应这类视频绝大多数用户都只是刷到看一眼就走，没有深入讨论的欲望，只是作为空闲时随意的消遣。
  ![alx text](chart\结果分析部分\视频基础信息\日常生活_视频基础信息.png)
- 4.文娱类视频（包括与电影电视剧，音乐，舞蹈，各国文化传播，动漫，(虚拟)主播等相关的视频）参与度较低。因为会存在反复观看的情况，会将各种参与度拉低，且不同视频参与度差别较大，会使参与度计算不准确。但对于这种情况，从收藏量较其他类型的视频遥遥领先可以看出，这种事视频用户喜爱度不会很低。
  ![alx text](chart\结果分析部分\视频基础信息\文娱_视频基础信息.png)

### 总体结论

视频评论区是否呈现“少数用户贡献多数评论”的二八结构，**并不取决于评论总量大小**，而取决于评论行为是否具备持续性、互动性与立场性。
当评论区演化为观点博弈与舆论竞争场域时，二八定律往往成立；反之，当评论仅承担一次性表达功能时，评论分布更趋均匀，二八定律不成立。

#### 满足“二八定律”的情形

在以下类型的视频评论区中，二八定律通常成立，且集中度往往明显高于“80/20”的经典比例：

- **游戏类视频**
- **新兴文娱类内容**（如新生代明星、饭圈文化、动漫、爆火音乐等）

其主要特征包括：

- 前 5%–20% 的用户往往贡献 60%–85% 以上的评论；
- 个别视频中，前 1% 用户即可贡献超过 70% 的评论；
- 洛伦兹曲线高度弯曲，基尼系数显著偏高。

形成原因主要包括：

1. 视频内容本身具备明显的争议性或阵营对立；
2. 评论区存在高频互动（争论、回复、反驳）；
3. 用户发表评论具有明确目的，如说服他人、强化立场或影响舆论走向。

在此类评论区中，评论行为已不再是简单的信息反馈，而是一种持续的、策略性的表达行为，客观上容易形成由少数高活跃用户主导的舆论结构。

#### 不满足“二八定律”的情形

在以下类型的视频评论区中，评论分布整体较为均匀，二八定律通常不成立：

- **新闻类视频**
- **日常生活类视频**
- **部分经典文娱内容**（如经典影视、音乐、传统文化）

其主要特征包括：

- 前 20% 用户平均仅贡献约 30% 左右的评论；
- 大多数用户仅发表 1 条评论；
- 评论参与结构接近均匀分布。

其根本原因在于：

1. 评论动机以情绪表达或态度表态为主；
2. 评论行为多为一次性行为，缺乏持续互动；
3. 评论区缺乏明显的立场对立或博弈空间。

在此情形下，评论区更多承担“情绪出口”或“存在感确认”的功能，而非讨论或舆论竞争平台。

#### 文娱类视频的两种评论区形态

文娱类视频的评论区结构呈现出明显的内容依赖性，可分为两种典型形态：

1. **新兴文娱内容**

   - 评论区高度集中；
   - 少数用户可主导大部分讨论；
   - 明显符合甚至远超二八定律。
2. **经典文娱内容**

   - 评论分布相对均匀；
   - 少量核心用户进行正常讨论；
   - 评论数量整体较少，不具备舆论操纵特征。

该差异主要源于是否存在饭圈文化、身份认同及立场站队等社会属性。

#### 关于评论区舆论主导的补充说明

需要指出的是：

- 二八定律成立并不必然意味着存在恶意操纵；
- 二八定律不成立也不等同于评论区“完全健康”。

但当以下特征同时出现时，可合理怀疑存在人为引导甚至操纵舆论的可能：

- 极少数用户进行异常高频评论；
- 评论内容高度重复、同质化；
- 该现象在大量同类视频中反复出现；
- 多出现在争议性或对立性极强的话题中。

#### 总结性结论

综上所述，**视频评论区是否满足“二八定律”，本质上取决于视频内容所激发的用户互动模式**。
当评论区以信息接收和情绪表达为主时，用户参与度趋于均匀，二八定律不成立；
当评论区演化为观点博弈与舆论竞争场域时，极少数高活跃用户将迅速占据话语主导地位，评论结构显著符合甚至远超二八定律。

## 5. 研究局限性与数据偏差

尽管本项目在数据获取与处理过程中尽量遵循规范化与稳健性的原则，但仍不可避免地存在一定的研究局限性与潜在数据偏差，需在解读分析结果时予以说明。

* **选取的视频无法代表一般情况**：由于对评论分析要求较大的数据才能看出较为准确的趋势，这里选取的都是播放量较高、评论数较多的视频，本身具有一定的特殊性，并不能代表一般情况。因此对这些视频的分析与结论是否真的能直接推广到同类视频还有待研究。
* **评论数据覆盖不完全**：对于评论量较大的视频或文章，受限于接口访问频率、平台风控策略以及实际运行环境，评论数据可能无法做到 100% 全量获取。部分较晚发布或位于深层分页的评论，可能未被纳入统计范围，从而对评论总量及用户活跃度的估计产生一定影响。
* **平台内容治理机制的影响**：B 站对评论区实行一定的内容管理与审核机制，包括但不限于删除违规评论、折叠争议性评论等。这意味着通过接口或页面获取到的评论数据，可能已经是平台筛选后的结果，实际历史评论数量与内容分布可能与获取数据存在差异。
* **评论形式的分析局限**：在文本分析过程中，本项目主要关注文字内容本身，对于仅包含表情符号、简短符号或高度依赖语境的评论，情感与语义识别能力有限。这类评论在数据中可能被低估其表达强度或情绪倾向，从而对整体情感分析结果产生一定偏差。
* **用户行为代表性的限制**：评论区用户本身并不等同于内容的全部受众，分析结果更多反映的是“参与讨论的活跃用户”特征，而非所有观看或阅读该内容用户的整体态度。因此，研究结论不宜直接外推至平台全部用户群体。

上述局限性并不否定数据分析的参考价值，但提示在解读结果时应结合平台机制与数据来源特点进行谨慎分析。

## .6 伦理与数据使用说明

本项目的数据获取与分析仅用于学习与研究目的，旨在从宏观层面探讨评论区讨论结构与用户参与特征，不涉及对具体个人的识别、追踪或评价。在项目实施与结果呈现过程中，遵循以下伦理与数据使用原则：

* **仅使用公开可见数据**：项目所使用的数据均来源于平台公开展示的信息，不涉及任何需要绕过权限或破解机制才能获取的内容。
* **避免个人层面的分析与指认**：分析过程关注评论数量分布、时间变化及整体特征，不针对单个用户进行行为评估、立场判断或标签化描述，相关结果仅在统计意义上讨论。
* **对潜在隐私信息进行处理**：在数据整理与存储过程中，对用户昵称、ID 等字段仅作为去重或计数用途使用，不在分析结果或可视化中直接呈现个人身份信息。
* **尊重平台规则与内容治理机制**：在数据获取过程中尽量遵循平台的访问节奏与使用规范，当接口或页面返回访问限制时，主动停止自动化请求，避免对平台正常运行造成影响。
* **谨慎解读与传播分析结论**：项目结论仅反映所选样本与特定时间范围内的讨论特征，不代表平台整体舆论倾向，也不应被用于对具体群体或个人进行价值判断。
